# Overview
- 사이트는 실시간 채팅(커뮤니케이션)과 간단한 게임(룰렛, 그래프게임) 제공
- 게임머니(포인트)는 어드민이 회원가입한 사용자에게 지급/차감
- 채팅은 부가 기능, 메인 서비스는 게임

# Core Features
## 1. 회원 시스템
- 회원가입/로그인(이메일, 닉네임, 비밀번호)
- 닉네임/이메일 중복 방지
- 어드민 계정 구분

## 2. 게임머니 관리
- 회원별 게임머니(포인트) 잔고
- 어드민이 회원에게 머니 지급/차감(수동 지급 UI)
- 머니 내역(지급/차감/베팅/당첨 등) 로그

## 3. 게임 기능
### 3.1 룰렛 게임
- 베팅 금액 입력, 룰렛 돌리기, 결과(당첨/꽝) 표시
- 당첨 확률/배당률 설정(관리자)
- 베팅/당첨 내역 기록

### 3.2 그래프 게임
- 실시간 그래프(랜덤 곡선) 표시
- 베팅 후 원하는 시점에 '스톱'하여 수익 확정(미리 멈추지 않으면 실패)
- 게임 진행/결과 실시간 반영
- 베팅/수익 내역 기록

## 4. 채팅
- 사이트 내 실시간 채팅(회원만 사용 가능)
- 관리자 메시지 구분
- 현재 접속자 수 표시

## 5. 관리자(어드민) 기능
- 회원 목록/검색/정렬
- 회원별 게임머니 지급/차감
- 게임별 통계(베팅액, 당첨률 등)
- 게임 설정(룰렛 확률, 그래프 파라미터 등)

# Roadmap
1. 회원가입/로그인/어드민 구분
2. 게임머니 관리(지급/차감/로그)
3. 룰렛 게임 구현
4. 그래프 게임 구현
5. 채팅(회원만) 및 접속자 표시
6. 관리자 UI(회원 관리, 게임 설정, 통계)
7. UI/UX 고도화, 모바일 대응, 배포 자동화

# Technical Architecture
- Node.js + Express + socket.io 서버
- DB: SQLite, MongoDB, 또는 MySQL(회원/머니/로그/게임내역)
- 프론트엔드: HTML/CSS/JS(React 권장), 반응형
- 실시간 통신: socket.io
- 배포: Docker, Cloudflare Tunnel, HTTPS

# Development Roadmap
- 1단계(MVP):
  - 실시간 채팅 송수신
  - 메시지 기록 전체 공개
  - admin 메시지 구분 표시
  - 서버 호스팅 및 배포
- 2단계(확장):
  - 메시지 영속화(DB)
  - 사용자별 닉네임 추가
  - 메시지 삭제/수정 등 부가 기능

# Logical Dependency Chain
- 서버/클라이언트 기본 구조 → WebSocket 연결 → 채팅 입력/기록 기능 → admin 구분 표시 → 서버 호스팅/배포

# Risks and Mitigations
- 서버 재시작 시 메시지 유실: MVP는 메모리, 추후 DB 연동 고려
- admin 인증: MVP는 간단한 방식(예: URL 파라미터), 추후 보안 강화
